Tiny Redis

Tiny Redis is a Redis driver for the D programming language. It is intentionally minimal, fast, simple, intuitive, feature complete, unit-tested, forward compatible, has no dependencies and makes working with Redis trivial.

It supports all of Redis's data types; keys, hashes, lists and sets. It also has functions for pipelining and transactions.

Quick Example

import tinyredis.redis; 

auto redis = new Redis("localhost", 6379);

redis.send("SET name Adil");
writeln("The name's ", redis.send("GET name")) //The name's Adil

Documentation

Tiny Redis has a very shallow learning curve (like most redis drivers). You learn best by using it. Documentation is included explaining the API, accessing results and usage examples.

  • Primary documentation is on the Wiki
  • Example.d - This file is included in the source download. It gives an overview of a variety of examples of calling commands, pipelining, doing transactions and handling/printing/iterating over results.
  • You may also want to check out the unit tests in tinyredis/*

Compilation Instructions

Static library

Download the shared static library for Linux and compile it with your source files. Ex:

dmd YOUR_SOURCE.d libtinyredis.a

For Linux x64, use the 64 bit version

dmd YOUR_SOURCE.d libtinyredis_x64.a

Source

Download the source code and compile tinyredis/* with your source files. Works on all platforms. This is recommended if you're new to TinyRedis. Ex:

dmd YOUR_SOURCE.d tinyredis/*

Other programs

There are few more programs included in the package as described below. Before running any of them, ensure you have the Redis server running on "localhost:6379" (those are the default Redis settings)

The Example Program

This program is most useful when read as source. It shows the various ways to use Tiny Redis. Run :

make example

The output is not interesting, look at the code.

The Interactive Console

The interactive console works like redis-cli. It is meant to show the simplicity of the driver, and is not a replacement for redis-cli. Run :

make console

The Benchmarking Program

A simple benchmark is included. To see how redis performs on your PC/Server, run :

make benchmark

Unit Tests Included

make test

Dependencies

None. Tested with dmd 2.059/2.060 on Linux x64.

Limitations

  • Lua Scripting has not been tested yet.
  • Pub/Sub and the "monitor" command need special support. Not done yet.
  • Supports D2 only
