By Adil Baig

Introduction

Tiny Redis is a Redis driver for the D programming language. It is intentionally minimal, fast, simple, intuitive, clean, unit tested, forward compatible, has no dependencies and makes working with Redis trivial.

It supports all of Redis's data types; keys, hashes, lists and sets. It also has functions for pipelining and transactions.

Tiny Redis is ..

intentionally minimal

With Tiny Redis you leverage your knowledge of Redis commands transparently. This minimizes the ramp-up time from learning the library to writing your application specific code. It's also simpler to build libraries on top of it.

fast

simple local benchmarking suggests TinyRedis can do 13,000 requests/second. With pipelining the numbers are orders of magnitudes higher. In practice, TinyRedis is fast enough to handle most real-world volumes. Watch the benchmarking space below.

simple

Run any Redis command using one simple call : 

Response response = redis.send(string)

That's it. All the other features reduce (more or less) boiler-plate from your code.

intuitive

By leveraging D's magic op* functions, TinyRedis aims to do what you expect.

clean

The encoder is separate from the parser, which is separate from the driver. In theory, we can mix and match and possibly introduce more backends easily.

unit tested

The parser and the driver have unit tests that ship with source.

forward compatible

Due to Tiny Redis' minimalist approach, any new commands introduced by Redis can be queried using the same API (as long as the protocol remains intact).

has no dependencies

A modern D2 dmd compiler is all you need.

makes working with Redis trivial.

Nuff said. Instructions are below.

Quick Example

Without further ado : 

import tinyredis.redis; 

auto redis = new Redis("localhost", 6379);

redis.send("SET name Adil");
writeln("The name's ", redis.send("GET name")) //The name's Adil

Compilation Instructions

Make a static library (Linux)

Download the source code and run `make`

make lib

Now, compile your source files using this library. Ex:

dmd YOUR_SOURCE.d libtinyredis.a

Source

Download the source code and compile tinyredis/* with your source files. Works on all platforms. This is recommended if you're new to TinyRedis. Ex:

dmd YOUR_SOURCE.d tinyredis/*

Other programs

There are a few more programs included in the package as described below. Before running any of them, ensure you have the Redis server running on "localhost:6379" (those are the default Redis settings)

The Example Program

This program is most useful when read as source. It demonstrates the API. Run :

make example

The output is not interesting, look at the code.

The Interactive Console

The interactive console works like redis-cli. It is meant to show the simplicity of the driver, and is not a replacement for redis-cli. Run :

make console

The Benchmarking Program

A simple benchmark is included. To see how redis performs on your PC/Server, run :

make benchmark

Unit Tests Included

make test

Dependencies

None. Tested with dmd 2.059/2.060 on Linux x64.

Limitations

  • Pub/Sub and the "monitor" command need special support. Not done yet.
  • Supports D2 only.
